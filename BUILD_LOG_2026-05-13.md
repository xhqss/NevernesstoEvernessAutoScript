## 一、项目定位

ok-nte 是基于 ok-script 框架的《异环》(Neverness to Everness) 游戏自动化工具。  
al-script 是一个通用游戏自动化框架（Alas 架构 + ok-script 设计），强制 1280×720 分辨率，不包含任何游戏特定逻辑。

NevernesstoEvernessAutoScript 是构建在 al-script 之上的游戏特定上层项目，包含 ok-nte 的全部游戏逻辑（战斗、钓鱼、音游、日常、角色、声音触发等），适配到 al-script 的 API 和架构。

## 三、构建过程

### 第一轮：快速搭建 (被否决)

- 创建 `src/` 目录，将所有文件平铺在内
- 子代理生成 CombatCheck/BaseCombatTask 等文件
- **问题**：结构不合 al-script 规范，UI 为桩代码，战斗系统非忠实移植

### 第二轮：nte/ 重构 (被否决)

- 重组为 `nte/` 包，对齐 al-script 的子目录命名
- 写入 NTEInteraction、BaseNTETask、CombatCheck、BaseCombatTask
- 子代理写入 sound、char、task 文件
- **问题**：`config/` 下放了 Python 代码，`module/` 下只有空文件夹，结构仍不对

### 第三轮：module/neverness/ 重构 (当前版本)

- 参考 Alas-with-Dashboard 的 `work.md` 确定正确架构
- **核心原则**：
  - `module/` — 所有 Python 实现代码
  - `config/` — 仅用户 JSON 配置文件，不放 Python 代码
  - `module/config/` — Python 配置系统代码
  - 功能子模块直接放在 `module/` 下（如 `module/campaign/`、`module/char/`）
- 创建 `module/neverness/` 作为游戏模块根
- 3 个子代理并行写入：任务系统(13文件)、角色/场景/声音系统(15+文件)、顶层文件

## 四、关键适配点

### 4.1 API 映射 (ok-script → al-script)

| ok-script | al-script |
|-----------|-----------|
| `ok.BaseTask` | `module.task.base_task.TaskBase` |
| `ok.TriggerTask` | BaseNTETask + trigger_interval 属性 |
| `ok.Box` | `module.feature.box.Box` |
| `ok.Feature` | `module.feature.feature.Feature` |
| `ok.og` (全局单例) | `module.neverness.globals.Globals()` |
| `ok.ConfigOption` | 字典 + config.py |
| `ok.Logger` | `logging.getLogger(__name__)` |
| `ok.color_range_to_bound` | `module.util.color.color_range_to_bound` |
| `ok.find_color_rectangles` | `module.util.color.find_color_rectangles` |
| `ok.CannotFindException` | `TimeoutError` |
| `ok.get_path_relative_to_exe` | 本地 `_get_assets_path()` |
| 分辨率 2560×1440 | 强制 1280×720 (al-script 自动缩放) |

### 4.2 保持原版的模块

| 模块 | 移植方式 |
|------|---------|
| NTEInteraction | 完整移植：operate/BlockInput/光标管理/PostMessage点击 |
| CombatCheck | 忠实移植：LV 形状指纹匹配、HSV 血条检测、异步战斗检测、CombatSettle 结算机制 |
| BaseCombatTask | 忠实移植：角色加载/切换/元素环/冻结追踪/协奏系统 |
| FishingTask | 忠实移植：HSV 拉力条检测（绿/黄/红色域）、长按/点按控条模式、蓝色咬钩检测、自动补饵卖鱼 |
| RhythmTask | 忠实移植：亮度阈值鼓点检测、异步生产者-消费者按键、结算/选歌界面检测 |
| SoundListener | 忠实移植：Butterworth 高通滤波、环形缓冲区、互相关匹配 |
| CustomChar | 忠实移植：AST 出招表语法解析器、if_ 条件指令 |
| CustomCharManager | 忠实移植：JSON DB 管理、特征图缓存与缩放匹配、Schema 迁移 |
